New Features

Direct asset linking from search results

Assets that have no metadata file in the same repo_loc are now directly linked to in the search results if the search matches an asset's tag(s) or transcript. This was not possible in the legacy Acumen, as assets could not be directly linked to, unless there was a metadata file in the same repo_loc.

Search box improvements

Session based previous search

Text in the search box will now only be saved for each session. This means that opening a new tab/browser and going to Acumen will no longer cause your previous search to auto-populate the search box's text field.

This type of session means the search box will clear if a users refreshes their browser by pressing F5

URI Aware

The search box in now aware of the URI. Linking to search results will not auto-populate the search text and select the correct search category (e.g., if you send a search for 'tall bike' in 'images', the search box will select the images category and the text box will be populated with tall bike).

Improved image search

The image search category has been set to also search based on genre for photographs or "picture postcards" (see section Search Categories tab).
